# 项目安装部署说明

## 1. 运行环境要求

### 后端
- **JDK**：17 或更高版本
- **Maven**：3.8+
- **MySQL**：8.x（可选，默认使用 H2 内存数据库）

### 前端
- **Node.js**：18+
- **npm**：9+

## 2. 启动后端服务

### 方式一：使用 H2 内存数据库（推荐，无需安装 MySQL）

```bash
# 进入后端目录
cd backend

# 编译并启动
mvn spring-boot:run
```

服务启动后访问：
- **API 地址**：http://localhost:8080
- **H2 控制台**：http://localhost:8080/h2-console （JDBC URL: jdbc:h2:mem:dietdb）

### 方式二：使用 MySQL 数据库

1. 创建数据库：
```sql
CREATE DATABASE diet_assistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 修改 `backend/src/main/resources/application.yml`，将 H2 配置注释掉，取消 MySQL 配置的注释：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/diet_assistant?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=utf8mb4
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: root
  jpa:
    hibernate:
      ddl-auto: update
    database-platform: org.hibernate.dialect.MySQLDialect
```

3. 启动服务：
```bash
cd backend
mvn spring-boot:run
```

### 方式三：直接运行 JAR 包

```bash
cd backend
mvn clean package -DskipTests
java -jar target/diet-assistant-1.0.0.jar
```

## 3. 启动前端

```bash
cd frontend
npm install   # 首次运行需要
npm run dev
```

前端默认在 http://localhost:5173 启动。

> 注意：前端开发服务器已配置代理，API 请求会自动转发到后端 8080 端口。

## 4. 访问应用

1. 确保后端已启动（端口 8080）
2. 确保前端已启动（端口 5173）
3. 浏览器打开：http://localhost:5173

## 7. 常见问题

**Q: 后端启动时报端口被占用？**
A: 修改 application.yml 中的 server.port 为其他端口。

**Q: Maven 下载依赖失败？**
A: 确保网络连接正常，pom.xml 已配置阿里云镜像仓库。

**Q: 前端页面无法加载数据？**
A: 检查后端是否已启动，以及前端 vite.config.js 中的代理配置是否正确。

## Backend configuration notes

The committed backend config does not store local database passwords. Use environment variables for MySQL:

```bash
DB_URL=jdbc:mysql://localhost:3306/diet_assistant?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8
DB_USERNAME=diet_user
DB_PASSWORD=your-local-password
mvn spring-boot:run
```

`spring.sql.init.mode` defaults to `never`, so `init.sql` is not rerun on each MySQL startup.
If a disposable local H2 database is preferred, start with the dev profile:

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

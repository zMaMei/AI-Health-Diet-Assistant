## 2. 数据库准备

### 2.1 安装 MySQL

确保 MySQL 8.x 已安装并运行。Windows 用户可下载 [MySQL Community Server](https://dev.mysql.com/downloads/mysql/)，macOS 用户可使用 `brew install mysql`。

### 2.2 创建数据库

登录 MySQL 并创建数据库：

```sql
CREATE DATABASE diet_assistant CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.3 配置数据库连接

编辑 `backend/src/main/resources/application.yml`，确认数据库连接信息与你的环境一致：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/diet_assistant?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: 你的MySQL密码
```

> **安全提示**：生产环境请勿将密码硬编码在配置文件中，建议通过环境变量传入：
> ```bash
> export DB_URL=jdbc:mysql://localhost:3306/diet_assistant?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=UTF-8
> export DB_USERNAME=root
> export DB_PASSWORD=你的密码
> ```

### 2.4 初始化表结构和数据

**方式一：JPA 自动建表**

首次启动时，后端 JPA 会自动根据实体类创建表结构（`application.yml` 中 `ddl-auto: update`）。自动建表后需手动执行 `backend/src/main/resources/data.sql` 插入演示数据，或：

**方式二：手动初始化（推荐）**

执行 `backend/src/main/resources/init.sql`（含完整 12 张表建表语句 + 演示数据）：

```bash
mysql -u root -p diet_assistant < backend/src/main/resources/init.sql
```

> **注意**：`init.sql` 执行后会自动创建演示用户、20 种食物、65 道食谱、示例饮食记录和默认预警规则，无需重复执行。

### 2.5 配置 AI API Key

编辑 `backend/src/main/resources/application.yml`，配置阿里云百炼 API Key：

```yaml
dashscope:
  api-key: 你的DashScope_API_Key
```

> 如未配置有效的 API Key，AI 功能（拍照识别、语音解析、食谱推荐、AI 对话等）将不可用，系统会自动降级为本地兜底逻辑。

## 3. 启动后端服务

```bash
# 进入后端目录
cd backend

# 首次运行需下载依赖
mvn install

# 启动服务
mvn spring-boot:run
```

服务启动后：
- **API 地址**：http://localhost:8080
- **健康检查**：http://localhost:8080/api/user-profile（需带认证 header，用于验证后端正常运行）

### 打包运行（可选）

```bash
cd backend
mvn clean package -DskipTests
java -jar target/diet-assistant-1.0.0.jar
```

---

## 4. 启动前端

```bash
cd frontend
npm install    # 首次运行需要
npm run dev
```

前端开发服务器默认在 http://localhost:5173 启动。

> 前端已配置 Vite 代理：`/api` 请求自动转发到后端 `http://localhost:8080`，无需额外配置反向代理。如需修改后端端口，同步更新 `frontend/vite.config.js` 中的 proxy 配置。

---

## 5. 访问应用

1. 确保 MySQL 服务已运行

## 8. 常见问题

**Q: 后端启动时报"Access denied for user"？**
A: 检查 `application.yml` 中的 MySQL 用户名和密码是否正确。

**Q: 后端启动时报"Unknown database 'diet_assistant'"？**
A: 需要先在 MySQL 中创建数据库，执行第 2.2 节的建库 SQL。

**Q: 后端启动时报端口被占用？**
A: 修改 `application.yml` 中的 `server.port` 为其他端口，并同步修改 `frontend/vite.config.js` 中 proxy 的 target 端口。

**Q: Maven 下载依赖失败？**
A: `pom.xml` 已配置阿里云镜像仓库。如仍有问题，检查网络连接或切换镜像源。

**Q: 前端页面无法加载数据？**
A: 检查后端是否已启动，以及 `vite.config.js` 中的代理配置是否正确。打开浏览器开发者工具查看 Network 面板。

**Q: AI 功能（拍照识别/语音解析/智能分析/推荐/AI 对话）无法使用？**
A: 确认 `application.yml` 中阿里云百炼 API Key 是否有效且额度充足。检查 DashScope 控制台确认 qwen-omni-turbo 模型可用。

**Q: 前端提示"请先登录"？**
A: 先进入"我的"页面，使用 demo/demo123 登录，或注册新账号。

**Q: 前端点击拍照/语音无反应？**
A: 确保浏览器已授予摄像头/麦克风权限。如使用 HTTP 访问，部分浏览器可能限制摄像头（建议使用 localhost）。

**Q: 语音录音后解析失败？**
A: 检查 `application.yml` 中 `dashscope.api-key` 配置是否正确。AI 不可用时语音解析返回模拟降级数据。

**Q: 表结构未自动创建？**
A: 检查 `application.yml` 中 `ddl-auto` 是否为 `update`。建议手动执行 `init.sql` 初始化。
